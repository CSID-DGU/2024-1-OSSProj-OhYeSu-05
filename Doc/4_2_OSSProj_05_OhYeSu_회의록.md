- 공유사항 및 차기 회의 일정
    - 매칭 알고리즘
        
        코사인 유사도 알고리즘으로 선정
        
        → 설문지 답변을 이진 데이터가 아닌 단위데이터(?)로 받는다고 가정.
        
        → 설문지로부터 수집한 사용자 특성을 벡터로 표현할 때, 각 특성의 중요도를 고려하여 벡터를 구성. 
             이후 벡터 간의 코사인 유사도를 계산하여 유사도를 측정. 
             코사인 유사도는 벡터의 방향을 고려하기 때문에 특성 간의 중요도를 고려할 수 있음.
        
    - 개발 툴
        
        Kotlin + Android Studio
        
        → 레퍼런스에서 이용한 언어와 툴을 기반으로 도메인 변경
        
        DB (MySQL+php 연동)
        
        → MariaDB로 선정
        
        API 
        
    - 차별점/차별화 방법
        
        도메인 변경 
        
        → (방탈출 매칭 → 룸메이트매칭)
        
        UI/UX 개선
        
        → 룸메이트 매칭 시스템이라는 도메인에 맞춘 UI/UX
        
        기능
        
        → 차단 설정/해제 기능 구현(해당 기능 추가 시 차단 해제/확인 기능도 필요)
        
        - 실시간으로 인스타 처럼 상대방을 차단할 수 있는 기능 도입
        - 차단 안했을 때/했을 때의 기능 변화
            - 채팅 가능 여부
            - 프로필 등장 여부
            - 차단 목록
            - 차단한 사람과 차단당한 사람이 보는 화면에 대한 기획
        
        → 알고리즘 변경
        
        - 코사인 유사도를 이용한 룸메이트 매칭 알고리즘
        - 차단 알고리즘 따로 구현X
            
            → 코사인 유사도를 통해 추천된 룸메이트 정보를 DB에 저장
            
            → 추후 룸메이트 차단시 DB의 정보를 불러와서 차단하는 방식으로 차단 기능 구현
            
        
        → 로그인 기능
        
        회원가입 기능 구현X, 회원정보를 DB에 미리 가지고 있고 로그인 하는 방식으로 구현(이클래스처럼)
        
        → 마이페이지
        
        회원 정보 수정, 개인 신상정보 수정, 설문지 수정, 차단유저 관리, 
        
        → 채팅 기능
        
        → 설문지 작성 기능
        
        DB 구성
        
        - 회원정보 데이터
        - 개인 신상정보 데이터
        - 개인이 희망하는 룸메이트에 대한 데이터
        - 차단한 룸메이트 db
        - 채팅방 db
        - 알고리즘
            - 유사도를 기반으로 추천된 룸메이트에 대한 데이터
