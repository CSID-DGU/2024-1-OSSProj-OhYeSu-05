#### 1.2.2 제품 개발
- **코드 작성**:
  - **목적**: 기능 명세서에 따라 시스템의 기능을 구현.
  - **작업**:
    - 백엔드 서버 개발 (Django 프레임워크 사용)
    - 프론트엔드 앱 개발 (Android - Kotlin 사용)

- **기능 구현**:
  - **목적**: 설계된 기능을 실제 코드로 구현.
  - **작업**:
    - 회원가입, 로그인 기능 구현
    - 매칭 알고리즘 개발
    - 룸메이트 추천 기능 개발
    - 룸메이트 매칭 기능 개발
    - 채팅 기능 개발
    - 차단 기능 개발
    - 신고 기능 개발
  - **산출물**:
    - 기능별 코드 ([Github 참고]("https://github.com/CSID-DGU/2024-1-OSSProj-OhYeSu-05"))

## 3. 이슈 관리
### 3.1 이슈 정의

| 제목 | 설명 |
| --- | --- |
| Feature | 기능 개발(새로운 기능 추가) |
| Test | 테스트 관리(테스트 추가, 테스트 코드 리팩터링, 프로덕션 코드 변경 없음) |
| BugFix | 버그 수정 |
| Docs | 문서 작성 및 수정(README.md 등) |
| API | 서버 API 통신 |
| Refactoring | 리팩토링(결과의 변경 없이 코드의 구조를 재조정함-가시성) |
| Comment | 주석 생성, 수정 및 삭제 |
| Question | 개발, PR관련 질문사항 |
| Setting | 개발 환경 세팅 |
| BackEnd | 백엔드 관련 이슈 |
| FrontEnd | 프런트 관련 이슈 |
| Chore | 설정 및 기타 이슈 |


### 3.2 이슈 회의
회의 진행 과정 공유
- **이슈 관리 회의**: 매주 월요일 수업시간 이후 (7PM ~)
- **회의록 작성**: [노션 회의록 작성](https://www.notion.so/f96bbd646a144726b02e791e5e1b8d23?pvs=4), 주요 논의 사항 기록

### 3.3 칸반 보드 활용
노션의 칸반 보드를 활용하여 이슈와 작업을 시각적으로 관리
- **KiriKiri Project 칸반 보드**:
  - **보드 링크**: [노션 칸반 보드](https://www.notion.so/4ebfd02a62dd428aad0df3da42971d07?pvs=4)
  - **카테고리 설정**: '할 일(To Do)', '진행 중(In Progress)', '완료(Done)' 등 카테고리 설정

  - **업데이트 및 공유**: 실시간 업데이트, 팀원과 공유하여 협업
